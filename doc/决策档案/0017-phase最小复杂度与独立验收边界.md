# ADR 0017：phase 最小复杂度与独立验收边界

> Status：已实现。启动闸门语义预审已审 phase 数量；vNext 运行时默认单 phase，旧 goal 不迁移。

phase 只表示可独立建检、且通过后会降低后续不确定性或解锁后续推进的交付里程碑；dgoal 默认使用一个 phase。普通实施步骤以及代码、测试、文档、验证等不同工作面优先作为 task，不得为展示计划结构或按技术层次机械拆 phase。代码改动、配套测试与相关文档默认是同一交付的任务，必须一起完成后再进入该 phase 的建检。启动闸门的 LLM 语义预审必须审查 phase 数量：任何额外 phase 都要证明其独立验收边界与推进价值，否则合并为 task 后才能进入用户确认；这不是可配置的复杂度偏好。已确认的旧 goal 保持其冻结 plan 与旧建检语义，不自动迁移或合并；用户要采用新规则，须清除后重新提案。这样避免每个细分步骤都引入一次独立建检，造成无必要的等待与 token 消耗；确有独立验收边界时仍保留多 phase 的建检价值。
