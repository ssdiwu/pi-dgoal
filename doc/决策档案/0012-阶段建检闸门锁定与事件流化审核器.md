# ADR 0012：阶段建检闸门锁定 + 事件流化审核器

阶段建检不过、无结论或审核器异常时，goal 继续保持 `active`，但进入**建检闸门锁定**：agent 只能围绕当前 phase 修正、补证据并重试 `dgoal_check`，**不能越阶段推进，也不能越终审推进**。这里拦的是推进规则，不是对所有工具做广义副作用审查；是否越界按**意图边界**判断，不按文件归属判断。独立审核子进程同时从“最终报告导向”改为“事件流导向”：父进程显式消费 `thinking_*`、`toolcall_*`、`text_delta`、`message_end` 等事件，投影成正式的**建检活性状态**给主 agent 与 TUI；`timeout` 不再表示“还没审完”，只保留给子进程、事件流或模型调用层面的真实异常（`auditor_error`）。本轮不引入自动换模型/备用模型，审核器继续复用当前会话模型；未来若要做模型容灾，另开版本讨论。
