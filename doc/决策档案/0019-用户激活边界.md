# ADR 0019：用户激活边界

> Status：部分被 ADR 0038 覆盖。Phase Plan / Goal Plan 仍保留本 ADR 的显式激活边界；Task Plan 改为 agent 可主动建立的日常默认。ADR 0034 的隐式例外已取消；ADR 0036 的自然语言显式入口继续有效。

`dgoal_*` 工具即使全局注册并对 AI 可见，也不授予 AI 主动启动或探测 dgoal 的权限。未获授权时，AI 可以建议用户使用 dgoal，但不得调用 `dgoal_propose`、`dgoal_plan`、`dgoal_check` 或 `dgoal_done` 来试探或开启流程。激活后各工具仍受既有 goal/phase 状态机约束。这样将 dgoal 保持为用户选择的执行模式，而不是 AI 自动接管的普通能力。

ADR 0036 的显式入口扩展：除 `/dgoal` 命令外，真实用户在空闲输入中以祈使句明确要求“使用/启动 dgoal”也会授予一次性显式启动权；冷会话的 `dgoal_propose` 据此建立 pending goal，但仍须经过语义预审与用户确认 UI。能力问句、引用/讨论、否定句、处理中追加和 `source=extension` 不授权；Pi input transform 的全权限信任边界详见 ADR 0036。

ADR 0034 的受限例外：仅当用户在全局配置显式开启 `implicitFinalOnlyStart`，LLM 才可对满足轻量、安全、可独立验收条件的任务隐式创建 `final_only` 有界 goal；项目配置不能授予此权限。
