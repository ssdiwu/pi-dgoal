# ADR 0024：src 模块地图与入口组装

> Status：已实现（vNext，含收敛说明）。

保留 `index.ts` 作为 Pi 注册、事件订阅和模块组装入口；按真实职责惰性迁入 `src/plan`、`src/audit`、`src/isolated-pi`、`src/tui`、`src/runtime`。`isolated-pi` 只承载隔离 Pi 调用、JSON 事件、模型注册表预检和候选选择的公共事实；auditor 与 context summarizer 各自保留不同重试和失败语义。审计用量账本归 `src/audit`（`usage.ts`），实际审核模型与 Goal Repair 展示归 `src/tui`。

**收敛说明**：原草案列出 `src/goal-runtime` 与 `src/startup` 两个目录。实际交付中，goal runtime（会话状态、续跑、计数器、终审反馈、终审修复账本）与 startup 编排（启动闸门、背景总结、propose 投递）仍合并在 `src/runtime`，因为当前只有一个消费者、未达到第二次分裂触发点（本 ADR 自身的「无需预建空目录 / 惰性迁入」原则）。未来若 runtime 体积或接缝压力再次升高，再按 0025 的边界拆出 `src/goal-runtime` 与 `src/startup`；届时本 ADR 追加修订记录。`src/runtime/README.md` 因此描述为「合并承载层」而非「过渡层」。
