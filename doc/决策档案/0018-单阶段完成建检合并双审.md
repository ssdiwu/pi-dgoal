# ADR 0018：单阶段完成建检合并双审

> Status：已被 ADR 0038 覆盖。三档 Plan 不再使用 `singlePhaseAudit` 合并凭据；Goal Plan 的单 phase 仍分别记录 phase check 与 goal check，完成状态由 `plan_update` 写入。

单 phase goal 不再先执行阶段建检、再执行目标终审。该 phase 的 task 已完成后，由一次独立的单阶段完成建检同时核验 phase 与 goal 的冻结验收条件；通过时原子地将 phase 记为 done 并记录统一 goal 审核凭据（`singlePhaseAudit`），但 goal 仍需 agent 调用 `dgoal_done` 关闭（不原子关 goal）；`dgoal_done` 消费该凭据，不重复终审。多 phase goal 仍保留逐 phase 建检和最终 goal 终审（含 phase(id)/goal/user_review 三路归因）：它们存在真实的中间交付与推进边界。此区分消除单 phase 下两次审核却语义重叠的等待与 token 消耗，同时不牺牲独立审核。
