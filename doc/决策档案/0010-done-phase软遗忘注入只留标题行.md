# ADR 0010：done phase 软遗忘——注入只留标题行，其下 task 软遗忘

`buildPlanContextBlock`（把 plan 注入 agent system prompt 的格式化函数）对建检通过的 done phase 只保留 phase 标题行，其下所有 task 的 subject 与 evidence 全部软遗忘——不再注入。对照百度 Unlimited OCR 提出的 R-SWA（Reference Sliding Window Attention，参考滑动窗口注意力）：goal + context 全局可见、每轮注入、不随进度退化，相当于"参考注意力"；当前 phase + 其下 task 相当于"工作记忆"；已完成 phase 的 task 细节相当于"已输出的 token"，应当软遗忘以聚焦当前进度。这条边界由三条事实共同决定：第一，done phase 的状态由 `dgoal_check`（建检）通过后显式标记（`setPhaseCompleted`），不是 agent 自封或聚合自动得出，因此 `ph.status === "done"` 是个可靠信号——只要显示 done 就一定经过建检验收，作为软遗忘触发器可信；第二，evidence 的权威来源是持久化的 `goal.plan`，不是注入层，建检/终审子进程读持久化全量、不读注入，agent 需要回查已软遗忘的 task 时靠两条天然路径——done phase 的标题行仍留在注入里（提供线索）、建检报告会注入给 agent（把问题拉回来）——所以不需要另建回查工具；第三，改动收敛到单函数 `buildPlanContextBlock` 一行分支（`if (isDonePlanStatus(ph.status)) continue;`），签名、返回结构、调用点、TUI 浮层渲染、建检子进程全部不受影响，风险可控。一个明确的边界是：软遗忘的时机是 phase 整体 done，不是单个 task done——当前正在做的 phase 里如果有已完成的 task，其 subject 和 evidence 仍然注入，因为 agent 还需要参照"已经做了哪些"来安排剩下的。本 ADR 不改持久化、不改建检可见性、不另建工具，只改注入投影层的格式化。
