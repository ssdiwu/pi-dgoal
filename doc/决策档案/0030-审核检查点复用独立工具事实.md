# ADR 0030：审核检查点复用独立工具事实

独立审核因超时、候选切换或 resume（恢复）重启时，不携带主 agent 会话历史，也不把上次模型自述当作证据。dgoal 改为持久化审核 child（子进程）`tool_execution_start/end` 产生的脱敏工具事实，按 phase/goal 范围和 workspace fingerprint（工作区指纹）隔离。只有同指纹下成功结束的精确命令可注入下一次 fresh context；running、failed、unknown 或工作区变化一律不复用。

这样保留独立审核的上下文隔离，同时避免重型验证从零重复。为防止长命令无限占用，阶段与目标审核分别采用跨候选共享的总时长预算。
