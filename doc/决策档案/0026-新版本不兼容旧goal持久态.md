# ADR 0026：新版本不兼容旧 goal 持久态

> Status：已决，待实现。

新版本不维护旧 goal 的运行兼容或行为版本分支。改用新的 dgoal custom entry type（持久化键）保存与加载状态，旧 `dgoal-state` entry 被新运行时忽略；用户升级后必须新开 `/dgoal`，不恢复、迁移、只读展示或自动清除旧 goal。这样避免两套建检、终审修复和启动语义在同一运行时共存带来的回归风险；发布说明必须明确此 breaking change（破坏性变更）。验证只覆盖新键写入/恢复与旧键不被误读，不维护旧状态机测试夹具；同时必须验证 npm 包包含 `src/`、RPC 加载、全量自动化、真实模型 smoke 与人工 TUI 复核。
