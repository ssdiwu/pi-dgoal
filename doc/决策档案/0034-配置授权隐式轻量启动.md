# ADR 0034：配置授权隐式轻量启动

> Status：已实现（v0.7.0）。条件式覆盖 ADR 0019：默认仍须用户显式激活；仅全局用户配置明确授权后，LLM 才可隐式启动。其“禁止执行本地测试/脚本”边界后由 ADR 0035 覆盖；其“显式激活只能输入 `/dgoal`”边界后由 ADR 0036 扩展为命令或自然语言明确要求；ADR 0037 进一步允许隐式 proposal 在只差确认授权时降级到普通显式确认。

## 背景

轻量任务即使适合 `final_only`，每次仍要求用户输入 `/dgoal`、审核 plan，增加了本应由 LLM 自己完成的计划整理与确认成本。用户希望在自己明确配置许可后，LLM 能判断任务足够轻量，直接建立 Task Plan 并执行；但不希望默认把 dgoal 变成 agent 自动接管普通任务的能力。

## 决策

- 新增全局用户配置 `implicitFinalOnlyStart`，默认 `false`。只有 `~/.pi/agent/pi-dgoal.json` 中显式设为 `true` 才授予 LLM 隐式启动权；项目 `.pi/pi-dgoal.json` 即使受信任也不能开启该权限，项目内容不得扩大 agent 的执行授权。
- 未开启时完整保留 ADR 0019：LLM 只能建议 `/dgoal`，不得因工具可见自行创建 goal。
- 开启后，LLM 仅在自己判断任务范围具体、可独立验收、没有高风险/不可逆外部动作且适合 `final_only` 时，才可从冷启动直接提交结构化 proposal 并开始执行；允许使用受边界校验的本地项目文件工具读写和读取外部网页/文档，但不允许通过 shell 执行任意 agent 可编辑脚本或测试代码，测试/脚本执行应转为显式 `/dgoal` 或独立审核子进程；禁止发送消息、发布/部署、`git push`、删除外部数据、修改账号/权限或消费付费资源。不符合目标/策略边界时拒绝；若唯一缺口是用户确认授权，则按 ADR 0037 降级显式确认，其他不确定情况不自动执行。
- 隐式启动只允许 `final_only` 和有界预算；默认基础上限为 24 个执行 turn、60 分钟有效运行时长、1 个终审修复回合，turn 宽限再给 24 个执行 turn；全局 `implicitFinalOnlyBudget` 可逐项覆盖这些安全默认值，墙钟宽限可显式设为 0。缺项或非法值仍回退到默认值并告警，绝不隐式放宽为无上限。`phased`、`unbounded`、无验收口、或需要用户取舍的任务仍须显式 `/dgoal` 与启动闸门审核。
- 隐式路径仍经过 proposal 结构校验与 semantic preflight；适合自动执行且通过时直接 active，不弹阻塞确认框；若显示计划即可补足授权，则降级为普通 pending goal 并弹同一确认 UI，不要求用户重输 `/dgoal`。
- 启动后立即以非阻塞通知说明“已按 `final_only` 自动建计划执行”，提供 `/dgoal s`、`/dgoal pause`、`/dgoal clear` 的可见退出入口；计划持续显示浮层照常出现。

## 后果

用户可选择让 dgoal 像带独立终审的轻量执行器一样工作，同时默认安装仍是显式授权、显式审核的保守模式。自动启动的范围被策略、预算、验收性和外部风险共同收窄；配置本身成为高权限事实，必须只从用户全局位置读取。

实现需在工具、system prompt 与运行时三处同时检查权限，不能只靠 prompt。需要覆盖：默认拒绝、全局启用后合格任务启动、项目配置无法越权、策略/预算越界拒绝、预审失败不启动、通知/UI 抛错不阻断正确状态，以及用户随时 pause/clear 的回归。
