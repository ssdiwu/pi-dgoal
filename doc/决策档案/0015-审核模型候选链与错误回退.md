# ADR 0015：审核模型候选链与错误回退

阶段建检与目标终审分别支持最多 3 个有序审核模型候选。审核器发生明确的运行错误或明确的纯文本配额耗尽时按候选顺序回退；业务 `REJECTED`、用户中断和 HTTP 400 不触发换模。候选全部耗尽后进入 `audit_error` 并暂停，不静默降级到当前执行模型。

## 背景

ADR 0014 让 `phaseAuditorModel` 与 `goalAuditorModel` 能固定两个审核范围的专用模型，同时明确不做备用模型。实际使用中，专用审核模型可能因认证、限流、服务端故障、模型下线或网络问题而不可用；直接回退当前执行模型又可能显著降低审核强度，让用户误以为仍由所配置的强审核器验收。

因此需要在“审核不中断”与“审核口径不静默降级”之间建立显式、可追溯的回退协议。

## 决策

### 1. 配置面

新增有序数组字段：

```json
{
  "phaseAuditorModels": [
    "openai-codex/gpt-5.6-sol:medium",
    "minimax-cn/MiniMax-M3:high"
  ],
  "goalAuditorModels": [
    "openai-codex/gpt-5.6-sol:xhigh",
    "minimax-cn/MiniMax-M3:high"
  ]
}
```

- 每个范围最多 3 个候选，超过时保留前 3 个并告警。
- 同一配置来源内，复数字段优先于对应单值字段，单值字段再优先于旧 `auditorModel`。
- 受信任项目级候选链整体优先于全局候选链，不合并两个来源。
- 复数字段为 `null` 时显式继承当前会话模型，并阻断同来源旧字段与更低来源。
- 空数组非法；数组内逐项校验，非法项按下标告警，合法项保序使用；重复项保留第一次并告警。若没有有效项，继续按配置优先级降级。
- 旧 `phaseAuditorModel`、`goalAuditorModel`、`auditorModel` 保持单候选兼容，不自动改写用户文件。
- 新建模板只展示 `phaseAuditorModels: null` 与 `goalAuditorModels: null`。

### 2. 模型可用性预检

候选不仅要格式合法，还必须能由与审核子进程同隔离边界的 Pi 模型注册表查询到。

- 先把完整值匹配为模型 ID；完整匹配失败后，才把末尾标准 `:thinking` 解析为思考等级。
- 预检使用结构化模型列表，不解析 `pi --list-models` 的人类可读表格。
- 结果在当前 Pi 进程内缓存；`/reload` 或新进程后重建。
- 预检自身失败时不误删候选，由真实审核运行时继续判定。
- 配置链最终没有有效候选时，可显式告警并降级到当前会话模型；这是配置错误降级，不是运行时候选耗尽。

### 3. 运行时回退

只有审核器技术异常触发候选回退：

- HTTP 401 / 403 / 404 / 408 / 429 / 5xx；
- 网络连接错误；
- 尚未产出明确审核结论时的零输出超时。

以下情况不触发换模：

- HTTP 400（视为请求格式、工具 schema 或兼容性问题，避免换模掩盖实现缺陷）；
- 用户中断；
- 已明确产出 `<APPROVED>` 或 `<REJECTED>`。

以下纯文本配额耗尽语义触发换模：`usage/plan/rate limit` 与 `reached/exceeded/hit/exhausted` 组合、`quota exceeded`、`insufficient quota`；普通 rate limit 描述、配置/元数据/字段/invalid/missing 等文本不触发。

有文本但缺少最终标记时，按“有部分输出的审核器异常”处理，不当作正式 `REJECTED`。

### 4. 部分输出与重试预算

- 有部分审核文本时，优先重试同一模型，最多 3 次。
- 重试时把已有文本作为临时 `<partial_audit_feedback>` 注入；它不是正式建检反馈，不写入 `phaseFeedbackById` 或 `finalFeedback`。
- 同一模型 3 次仍未形成明确结论，再携带部分审核文本切换下一候选。
- 零输出且命中可回退错误时，不消耗同模型 3 次预算，直接尝试下一候选。
- 部分反馈注入设置总长度上限，避免重试上下文无限增长。

### 5. 可观测性与耗尽行为

- 切换过程通过工具 `onUpdate` 实时展示“模型 A 失败 → 模型 B”。
- 最终工具 `details` 记录实际采用模型、配置降级状态，以及每次尝试的模型、结果与失败原因；轨迹随 session 保留，不扩充 `GoalState`。
- 错误轨迹不进入审核 prompt；只有部分审核文本会传给下一次尝试。
- 运行时所有候选均失败时进入 `audit_error` 并暂停，绝不静默回退当前执行模型。

这项决策反转 ADR 0014 “不引入自动换模、备用模型或容灾策略”的边界；ADR 0014 关于双审核范围、配置来源、信任边界、`null` 继承和模板初始化的其余结论继续有效。

## 理由

- 显式候选链能在审核模型临时不可用时继续验收，又不把较弱执行模型伪装成专用审核器。
- 只对技术异常回退，避免不同模型对业务结论投票或“审不过就换一个”。
- 部分反馈续传保留已完成的审查工作，同时限制同模型重试次数与候选数量，控制 token、等待时间和故障放大。
- 结构化模型预检与完整尝试轨迹让配置错误、运行时故障和最终审核口径都可独立复验。
