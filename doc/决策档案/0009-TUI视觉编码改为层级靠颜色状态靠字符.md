# ADR 0009：TUI 视觉编码改为“层级靠颜色，状态靠字符”

`/dgoal s` modal 和持续计划浮层统一采用“层级靠颜色，状态靠字符”的视觉语法：goal、phase、task 用固定层级基色区分；pending / in_progress / done / blocked 统一用前缀字符表达，不再让整行、前缀颜色或粗体承担状态语义。颜色选择以跨主题可见性为前提：必须同时兼顾浅色背景与深色背景，不能拿 `yellow`（黄色）这类在白底容易丢失的颜色当主语义。当前收敛到的安全集合是 `goal = accent + bold`、`phase = text`、`task = dim`。phase 与 task 统一使用同一套状态字符 `○ / ◐ / ✓ / ⚠`；goal 继续保留 `🎯` 作为标题锚点。done 的 phase 与 task 都允许加删除线，但删除线只作用于标题文本，不覆盖树形结构符号和状态字符。`activeForm`（进行中文案）与 `blockedReason`（阻塞原因）曾作为行内辅助信息弱化显示；ADR 0041 已移除前者，进行中由 task subject 后的动态省略点投影，`blockedReason` 仍不参与删除线，也不承担层级或状态主语义。这样在大量 pending 项、长计划或 phase/task 混排时，用户仍能一眼看出层级，避免当前按状态染色把 phase 与 task 一起压成一片 muted 的问题；本 ADR 覆盖 ADR 0008 中“emoji + 颜色按 status”的那部分视觉编码决定，但不改变其 top-center overlay modal 形态决策。

实施上采用分步落地：本次先改 `/dgoal s` modal，完整落实层级基色 + 状态字符；持续计划浮层先只统一状态字符、删除线和结构，不在本次升级为 theme-aware 彩色 widget。持续浮层彩色化单列到下一版本方案中处理，以减少本次引入新的 TUI 渲染 bug 面。
