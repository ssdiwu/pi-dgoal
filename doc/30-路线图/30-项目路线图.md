# 30 - 项目路线图

> dgoal 演进计划。当前架构见 `../10-架构与运行/`，关键决策见 ADR 0038，ID 语义见 ADR 0039，Description 与持久化 v2 见 ADR 0042。

## 当前阶段：ADR 0042 已实施，等待下一次破坏性发布

最近已发布版本为 v0.7.6。当前未发布主线在三档 Plan + 八个两词工具上增加三层必填 Description、删除 `contextSummary`，并把持久化升级为 `dgoal-plan-v2`；`dgoal-plan-v1` 活动状态不迁移。

### 已完成

1. `PlanType`、plan revision 与 phase/goal `CheckRecord` 数据模型。
2. 八个新工具的公共注册与核心状态守卫。
3. Task Plan 直接建立、整份替换、隐藏 phase 与 task-first TUI 投影。
4. Phase/Goal Plan 入口包装、显式启动闸门、语义预审与类型切换反馈。
5. check / update 职责分离：check 只记录，`plan_update` 唯一写完成。
6. 删除 startup 中的隐式动作护栏、runtime budget 消费与旧工具注册。
7. 新运行时测试：工具集合、Task Plan 生命周期、主动暂停、Goal Plan 双层审核链。
8. 当前架构文档、README、术语表与 ADR 0038。
9. 清理旧工具实现与 budget / implicit 死分支，迁移 RPC、smoke driver 和公共行为测试。
10. 自动验证通过：`npm test`、`npm run test:rpc`、`npm run test:smoke:runtime` 与 TypeScript 检查。
11. Phase/task 双 ID namespace：各自从 `1` 开始，task 在 Plan 内全局唯一（ADR 0039）。
12. 三层 Description 必填、goal 冻结与 phase/task 显式修订；执行 prompt 注入且 auditor 不把它升级为完成门。
13. 删除 `contextSummary` 全链路，使用 `dgoal-plan-v2` 严格隔离旧持久态。
14. `/dgoal s` 列表/详情两层 Modal：逻辑项选择、详情滚动、返回保位与 Task Plan 隐藏 phase。
15. Phase/Goal proposal 固定执行精简的端到端、生命周期/真实调用链、失败路径与验收契约一致性软检查；Task Plan 和 semantic preflight 职责不扩张。
16. `plan_read` 与 `/dgoal s` 共享当前 frontier 的 why-not 纯投影，并仅展示现有字段中的最新 check/反馈/完成声明；不新增历史、计数、耗时或恢复结构。

### 发布前复核

- `npm run test:smoke`：真实模型隔离 smoke，脚本已迁移；发布前按 token / 网络条件运行。
- 真实 TUI：启动确认、Plan 类型切换、持续显示浮层、`/dgoal s` Modal 与 rejected 修复交互。

以上是发布前复核，不作为本轮机器完成门；人工体验仍归用户复核项。

## 当前工具链

```text
建立：task_plan / phase_plan / goal_plan
管理：plan_create / plan_read / plan_update
审核：phase_check / goal_check
```

完成链见 `../10-架构与运行/10-建检循环与三层结构.md`。

## 历史里程碑

- **v0.2–v0.5**：goal / phase / task 三层、Task Plan reducer、持续显示浮层、启动闸门、阶段建检与 goal 终审。
- **v0.5.7–v0.6.4**：审核候选链、冻结独立验收契约、隔离 runtime、语义预审 idle timeout、审核检查点和 agent 主动暂停。
- **v0.7.0**：曾引入 `final_only` / `phased`、bounded/unbounded budget 与隐式轻量启动；这些组合现由 ADR 0038 覆盖。
- **v0.7.1**：自然语言显式启动与“轻提案、硬执行”；自然语言显式入口和语义职责分层继续保留。
- **v0.7.2**：Pi 核心包改为 peer dependencies，避免 npm 安装替换宿主依赖树。

详细版本时间线见 `../../CHANGELOG.md`，历史决策保持在 `../决策档案/`，不改写过去。

## 后续候选

- **Plan 运行计数**：在 `/dgoal s` 展示 task/phase/goal check 次数、最近审核耗时；先等真实可观测性痛点。
- **ObservableOperation 公共骨架**：语义预审和独立审核已有两套 idle-timeout；第三个同类场景出现后按三次原则抽取。
- **evidence 结构化**：当前是文本；真实需要稳定解析命令与预期输出时再设计。
- **check 历史可视化**：当前查询只展示最新 CheckRecord、反馈与完成声明；内部修复索引和审核检查点不展开。有真实逐次回看需求再扩展。

## 暂不做 / 远期

- 跨 session 后台 goal、多 Plan 池、定时/事件调度。
- 任意深度内容树；task 分解继续用 `blockedBy` DAG。
- 自动 Git commit、回滚、push、发布或删除。
- 替代项目自身测试命令。
- 为想象中的 provider / adapter 提前建接口。
