# v0.6.4 审核安全与主动暂停收尾方案

> 本方案负责把 v0.6.3 之后已经完成的主动暂停、审核检查点可靠性与敏感信息保护改动收敛为一个可发布版本。受 `30-路线图` 约束。

## 范围

- 新增 agent 主动暂停出口 `dgoal_pause` 与 `agent_blocked` 暂停语义。
- 修复长审核工具执行误超时、审核检查点复用边界与 child 事件结论校验。
- 加强 workspace fingerprint 的普通/ignored 文件变化与读取失败处理。
- 加强审核恢复上下文中的命令敏感信息脱敏。
- 同步架构文档、ADR、项目地图、测试地图和发布元数据。

## Checklist（检查清单）

### 功能与安全

- [x] `dgoal_pause` 工具、暂停原因边界、resume 语义和 UI fail-soft 回归通过。
- [x] 审核 child 的 start/end 工具名匹配，只有 `isError:false` 才形成成功 checkpoint。
- [x] Git、普通 untracked、ignored 文件或文件内容读取失败时禁止复用旧 checkpoint。
- [x] 命令 CLI、shell assignment、URL credentials、Cookie/session、HTTP header、URL query、PEM private key 敏感值脱敏。
