# 40 - 版本实施方案

存放当前版本级实施方案和验收记录。受 `30-路线图` 约束。

实施某个版本切片前，在此目录建方案文档（命名 `序号-版本号-方案名.md`）。序号只在本目录当前方案内排序；归档后可从 `41` 重新起排，避免当前目录被历史编号拖累。

## 当前方案

当前没有进行中的版本实施方案。v0.7.2 仅修复 npm 安装依赖边界，记录在 `CHANGELOG.md` 与 [`30-项目路线图.md`](../30-路线图/30-项目路线图.md)；v0.7.1 对隐式执行、自然语言启动与 proposal 语义职责的收敛由 ADR 0035–0037 记录，没有为已完成切片事后补建重复方案。

新版本切片开工前，应在本目录建立一个权威实施方案，并在方案内保留 checklist（检查清单）与验收记录。
